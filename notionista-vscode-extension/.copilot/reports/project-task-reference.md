## Task Sequence

Each project contains 5 sequential tasks:

| Code    | Order | Description                  |
| ------- | ----- | ---------------------------- |
| **T01** | 1     | Initial task / Setup         |
| **T02** | 2     | Development / Implementation |
| **T03** | 3     | Review / Refinement          |
| **T04** | 4     | Testing / Validation         |
| **T05** | 5     | Completion / Handoff         |

**Workflow:** Tasks are assigned daily and completed same-day. When T01 is done, T02 becomes the next task.

## Workflow Patterns

### Daily Workflow

1. **Query today's meeting** for the relevant team
2. **Get next task** from project (filter: `Done=false`, sort by `Task Code`, take first)
3. **Add task relation** to meeting's Action Items
4. **Complete previous task** (mark `Done=true`) if applicable
5. **Update task due date** to today

### Sprint Planning Workflow

1. **Query active projects** for the team (filter: `Status=Active`)
2. **Review project phase** and milestone alignment
3. **Assign projects** to sprint meeting via relations
4. **Track progress** via rollup formulas

### Task Completion Workflow

1. **Mark task Done** (`Done=true`)
2. **Create Portfolio page** (if deliverable exists)
3. **Link Portfolio** to originating Task, Project, Team
4. **Archive task** if needed (`Archived=true`)
